Uma API RESTful é uma interface de programação de aplicações (API) que segue os princípios e restrições do estilo arquitetural REST (Representational State Transfer).

Ela permite a comunicação entre sistemas diferentes na web de forma eficiente e padronizada utilizando protocolos nativos da web, principalmente o HTTP. Seus principais conceitos incluem:

  • Recursos: Tudo é mapeado como recursos identificados por URIs únicas (ex: /usuarios).
  • Métodos HTTP: Utilização correta dos verbos como GET (leitura), POST (criação), PUT (atualização) e DELETE (exclusão).
  • Stateless: Cada requisição contém todas as informações necessárias para ser processada, sem armazenar sessões no servidor.

No JavaScript moderno (ES6+), existem três maneiras de declarar variáveis com diferenças cruciais em escopo, hoisting (içamento) e reatribuição:

Palavra-chave Escopo Hoisting Reatribuição
var Função Sim (inicializado como undefined) Permite
let Bloco { } Não (fica na Temporal Dead Zone) Permite
const Bloco { } Não (fica na Temporal Dead Zone) Não permite

Dica: Prefira sempre usar const por padrão para evitar mutações acidentais, e use let apenas quando o valor precisar ser reatribuído.

Embora frequentemente usados juntos, Git e GitHub são ferramentas distintas com propósitos complementares no fluxo de trabalho de um desenvolvedor:

1. Git (O Sistema): É um software de controle de versão distribuído que roda localmente no seu computador. Ele serve para:

  • Registrar todo o histórico de alterações feitas nos arquivos de um projeto.
  • Facilitar a criação de ramificações (branches) para testar novas funcionalidades sem afetar a versão principal estável.
  • Reverter arquivos para estados anteriores de forma rápida e segura.

2. GitHub (A Plataforma): É um serviço de hospedagem web na nuvem para repositórios que utilizam o Git. Ele serve para:

  • Armazenar os repositórios Git externamente de forma segura.
  • Colaborar com outros desenvolvedores através de ferramentas como Pull Requests (propostas de alteração de código) e Issues (relato de bugs ou tarefas).
  • Automatizar fluxos de build, testes e deploy de aplicações (CI/CD).
